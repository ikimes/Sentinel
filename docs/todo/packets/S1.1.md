# Slice Packet: S1.1

## Goal

Add automated proof for legacy schema adoption of an old `EnsureCreated()` database into the migration baseline.

## Why Now

`Sentinel.Shared/Messaging/DatabaseBootstrapper.cs` contains real transition logic that is not directly proven by automated tests.

## Interfaces Affected

## Implementation Steps

1. Create a test case that provisions a database with the legacy schema shape and without `__EFMigrationsHistory`.
2. Start the API against that database using existing test infrastructure.
3. Assert that legacy adoption occurs successfully.
4. Assert that `actor_id`, `actor_type`, and `actor_display_name` are present on `masstransit.dispatch_records`.
5. Assert that the baseline migration row is recorded in `__EFMigrationsHistory`.
6. Keep the test narrow and deterministic.

## Acceptance Criteria

- A failing legacy-adoption path would fail the new automated test.
- The test proves baseline migration history is recorded.
- The test proves actor columns are present after adoption.
- No production code changes are introduced unless absolutely required to enable the test.

## Verification

- `dotnet test Sentinel.Api.Tests/Sentinel.Api.Tests.csproj`

## Purpose

This memo compresses the current `v2` position into the shortest useful strategic readout.

It is not a full package replacement.

It is the quickest way to understand what Sentinel now believes, what still looks fragile, and what must be tested next.

## What We Now Believe

### 1. Sentinel has a coherent product center now

The strongest current center is:

Sentinel is a narrow, evidence-grade overlay for one recurring cross-system regulated follow-up matter.

That is a much healthier position than broad control-plane language by itself.

### 2. The product identity is strongest as a workspace

Sentinel now looks most coherent when understood as:

- a governed matter workspace on the surface
- a coordination-state layer underneath
- a thin control-surface entry only when that helps the buying motion

### 3. The wedge is real only when the workflow is painful and cross-system

Sentinel is not for every regulated workflow.

It becomes strong when:

- one matter crosses teams and systems
- ownership and due-state become hard to see after intake
- evidence and downstream completion are fragmented

### 4. The product should copy discipline, not rhetoric

The best lessons from the analog set are:

- enter through one painful workflow
- earn evidence-grade trust
- prove value with minimal footprint
- coexist with incumbents
- land with a control surface only if it still points back to the deeper workspace product

## What Still Worries Me

### 1. Overlay buying path

The product may be useful and still unbuyable.

The open risk is not whether an operator sponsor exists.

It is whether that sponsor connects to:

- named budget
- approval authority
- acceptable overlay boundaries

### 2. Superiority over internal build

Internal build is still the most dangerous substitute.

Sentinel does not need to prove internal build is impossible.

It needs to prove buying Sentinel is more credible than rebuilding the matter model, evidence trail, and workspace inside ticketing, dashboards, and integrations.

### 3. Demand density remains important, but now it mostly feeds the two risks above

If the workflow is not frequent and painful enough, the overlay path gets weak and internal build starts looking more rational.

## What We Should Test Next First

### 1. Test the overlay buying path directly

We need to learn:

- who sponsors the first pilot
- who owns the budget
- who approves an overlay beside the incumbent stack
- what packaging makes the first purchase legible

### 2. Test the internal-build comparison directly

We need to learn:

- where internal build is genuinely strong
- where it drifts into tickets, reports, and services
- whether Sentinel's stable matter model and workspace really outperform it

### 3. Keep the first proof minimal while testing both

The MVP proof still needs to stay small:

- one source system
- one downstream target
- a small user group
- one recurring matter type

That is the smallest shape that can test both buying path and internal-build superiority honestly.

## Working Decision Frame

Right now Sentinel is:

- strategically coherent
- product-wise more distinct
- commercially unearned in two places above all others:
  - overlay buying path
  - superiority over internal build

The next phase should be judged by whether those two doubts get sharper, weaker, or stronger.

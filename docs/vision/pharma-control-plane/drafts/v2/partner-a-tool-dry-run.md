## Purpose

This exercise dry-runs the two new `v2` field tools against the stronger hypothetical prospect from `v1`.

The goal is not to pretend Partner A is a real customer.

The goal is to see whether the tools converge on the same strategic read:

- is the workflow worth pursuing
- is the buying path credible
- does Sentinel look stronger than internal build

## Partner A Summary

Partner A is the stronger hypothetical candidate from `v1`:

- multinational pharma sponsor
- incumbent safety platform already in place
- local affiliate workflows in key markets
- one downstream quality-adjacent target
- global and local teams both involved in recurring post-intake follow-up
- current coordination handled across system notes, email, spreadsheets, and status meetings

The prior `v1` read was:

- overall score: `38 / 50`
- qualification: `worth pursuing as a design-partner candidate`
- biggest weaknesses: buyer clarity, overlay appetite, and build-vs-buy pressure

## Tool 1: Overlay Buying Path Interview Rubric

### Scorecard

| Dimension | Score | Reason |
|---|---|---|
| `Workflow visibility to leadership` | `3` | pain is visible during exceptions and recurring meetings, but not yet clearly a top program |
| `Sponsor clarity` | `4` | one operations-oriented sponsor archetype is plausible: global safety operations or PV transformation leadership |
| `Budget clarity` | `3` | a plausible budget path exists through safety operations improvement or PV systems optimization, but it is still fragile |
| `Approval clarity` | `3` | likely approvers and blockers are visible: enterprise architecture, incumbent owner, security, procurement |
| `Overlay appetite` | `3` | a narrow overlay is plausible, but incumbent extension still feels like a credible alternative |
| `Incumbent resistance` | `2` | the incumbent safety-platform owner is a real likely blocker and overlap concerns are not yet neutralized |
| `Commercial wrapper fit` | `4` | narrow operational control overlay or evidence-grade follow-up workspace both feel legible as a first purchase |

### Total

`22 / 35`

### Interpretation

This lands in:

`promising but fragile`

The workflow is strong enough to support an overlay conversation, but the buying path is not yet earned.

The biggest weaknesses are:

- budget clarity
- incumbent resistance
- only moderate overlay appetite

### What This Tool Says

Partner A is a credible overlay-buyer candidate only if future discovery can close three gaps:

- name the actual pilot budget source
- make the overlay boundary easy for incumbent and architecture stakeholders to accept
- identify the first commercial wrapper that feels small without hollowing out the product

## Tool 2: Internal Build Comparison Scorecard

### Scorecard

| Dimension | Score | Reason |
|---|---|---|
| `Matter-model stability need` | `4` | the workflow crosses systems, teams, and markets often enough that a stable shared matter model looks important |
| `Evidence and provenance rigor` | `4` | fragmented evidence and decision reconstruction appear operationally meaningful |
| `Operator workspace gap` | `4` | operators need one place to understand owner, clock, blockers, and downstream state |
| `Downstream-state fragmentation` | `4` | one downstream system plus local/global handoffs create real manual reconstruction today |
| `Cross-system and cross-team breadth` | `4` | breadth is meaningful, though not yet sprawling |
| `Internal workflow team strength` | `3` | internal build is plausible, but not obviously dominant from the current facts |
| `Internal-build drift risk` | `4` | this workflow could easily drift toward tickets, reports, and custom stitching if built locally |
| `Time-to-proof advantage` | `4` | Sentinel's limited-footprint proof still looks lighter than a broad internal project |
| `Repeatability beyond the first workflow` | `4` | the matter shape appears reusable across repeated examples |
| `Buyer confidence in buying versus building` | `3` | there is an opening, but no decisive moat yet |

### Total

`38 / 50`

### Interpretation

This lands in:

`promising but not yet decisive`

The strongest part of Sentinel's position is product shape:

- stable matter model
- evidence-grade trail
- operator workspace
- limited-footprint proof

The weakest part is still whether a buyer would conclude those advantages are strong enough to buy rather than stitch together internally.

### What This Tool Says

Partner A is exactly the kind of account where Sentinel can make a serious case against internal build, but not yet an automatic win.

To become decisive, Sentinel would need stronger evidence that:

- the workspace is noticeably better in daily use than incumbent screens plus reporting
- the same matter model survives a second closely related workflow
- buying feels faster and safer than internal build, not merely cleaner in concept

## Where The Two Tools Converge

They agree on four important things:

### 1. The workflow is real enough to pursue

Neither tool says "walk away."

Both say the underlying operational problem is legitimate and commercially interesting.

### 2. Product fit is stronger than buying-path clarity

The internal-build score is better than the overlay-buying score.

That means Sentinel currently looks more coherent as a product than it does as a purchasing motion.

### 3. This is not an automatic first-customer yes

Both tools say some version of:

- worth pursuing
- not yet decisive

### 4. The main friction is commercial, not conceptual

Neither tool suggests the workflow is too vague or the product is too blurry.

Both point back to:

- budget path
- incumbent resistance
- buy-versus-build confidence

## Where The Tools Complement Each Other

They illuminate different parts of the same problem:

- the overlay rubric tests whether the account can actually buy a narrow overlay
- the internal-build scorecard tests whether Sentinel deserves to win even if it can be bought

That combination is useful because a prospect can fail in two different ways:

- buyable, but not distinct enough
- distinct enough, but not buyable

Partner A currently looks closer to:

`distinct enough to pursue, but not yet buyable enough to trust`

## Combined Verdict

Partner A remains a strong benchmark prospect for Sentinel.

The combined read is:

- `workflow qualification`: strong
- `overlay buying path`: promising but fragile
- `internal-build pressure`: promising but not yet decisive
- `overall`: good first-discovery target, not yet a proven first customer

## What A Real Prospect Would Need To Add

To upgrade this profile from strong benchmark to near-term first-customer candidate, a real prospect would need to provide:

- one named budget source
- one named sponsor with enough pilot authority
- one clearer answer for why overlay beats incumbent extension
- one clearer answer for why buying Sentinel beats local build

## Practical Next Step

Use Partner A as the benchmark profile for the next real-world dry runs.

If a real prospect scores materially lower than this on either tool, Sentinel should be cautious.

If a real prospect scores higher on both tools, the buying path may finally be getting earned.

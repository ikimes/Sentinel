## Required Edge Cases

Add explicit test coverage for:

1. `GET /api/compliance/check/{requestId}` returns `404` for an unknown request
2. status-by-id returns `failed` and includes ledger failure detail when a terminal failed row exists
3. recent/history semantics remain correct when duplicate dispatch rows exist for the same request
4. accepted-time semantics remain tied to the earliest dispatch row

## Implementation Steps

1. Extend `ComplianceApiTests.cs` with compact edge-case tests only.
2. Reuse the existing seed helpers where possible.
3. Keep the tests narrow and contract-focused.
4. Do not refactor production code in this slice unless a test reveals a genuine bug.

## Acceptance Criteria

- The four required edge cases are covered by automated tests.
- The tests fail if the contract regresses.
- No production files are edited unless a real bug blocks the slice.
